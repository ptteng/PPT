1.背景介绍

JSP页面里可以直接书写java代码，利用<% %>,直接在里面调用Javabean,就会造成很混乱的局面， EL的出现就是为了简化JSP页面,使得JSP页面的处理程序编写起来更加简洁，便于开发和维护。

jstl就是JSP标准标签库，jsp标签的集合，封装了大量的jsp应用的功能，我们自己也可以自定义标签， 将自己的常用功能封装起来直接用标签调用，也极大的简化的JSP页面。

2.知识剖析

EL表达式

1.简介

EL（Expression Language）表达式语言是JSP 2.0增加的技术规范，是一种简单的语言，提供了在JSP中简化表达式的方法， 目的是为了尽量减少JSP页面中的Java代码，使得JSP页面的处理程序编写起来更加简洁，便于开发和维护。

2.EL作用域对象

在JSP中访问模型对象是通过EL表达式的语法来表达。所有EL表达式的格式都是以“${}”表示。 例如，${name}代表获取变量name的值。 当EL表达式中的变量不给定范围时，则默认在page范围查找，然后依次在request、session、application范围查找。 也可以用范围作为前缀表示属于哪个范围的变量，例如：${pageScope.name}表示访问request范围中的name变量。

EL表达式

3.EL表达式输出

语法是${[作用域.]属性名[.子属性]}，例如${requestScope.Emp.age} el表达式支持直接将运算结果输出，也支持将绝大多数对象进行输出，本质是执行toString()方法。例如${emp.salary + 300},${1<=3 && 2>4}

4.EL输出参数值

EL表达式内置param对象来简化参数的输出，当请求体中有参数的时候可以用param和paramValues对象用来访问参数值，通过使用request.getParameter方法和request.getParameterValues方法。 例如传一个name参数，表达式是用${param.name}获取。 还可以用header和headerValues对象用来访问信息头，通过使用 request.getHeader方法和request.getHeaders方法。 例如${header.user-agent}，或者${header["user-agent"]}。

JSTL标签库

1.简介

jsp标准标签库(jstl),封装了jsp应用的通用核心功能，提供的功能分类为包括核心标签，格式化标签，sql标签，xml标签，jstl函数。JSTL标识库的作用是减少JSP文件的Java代码， 使Java代码与HTML代码分离，所以JSTL标识库符合MVC设计理念。由SUN（oracle）公司定义规范，由Apache Tomcat团队实现。

4.解决方案

1.el表达式和jstl的关联

EL表达式本身就是jstl1.0中技术，但从JSP2.0开始（jstl1.1)就分离出来作为JSP的标准，也就是相当于集成到jsp技术中，不需要专门引用。

2.param和paramValues对象，header和headerValues有何区别

param和header对象返回单一值，而paramValues和headerValues则返回一个字符串数组。

6.扩展思考

1.EL表达式中“.”和“[ ]”的区别

EL表达式提供"."和"[]"两种操作符来存取数据。两者通常情况是通用的：${user.sex}或者${user["sex"]}都是允许 的。此外，"[]"还可以用来进行集合元素中的定位：${booklist[0].price}。 以下两种情况是不可以通用的： 1)包含特殊字符 当要存取的属性名称中包含一些特殊字符，如 . 或 - 等并非字母或数字的符号，就一定要使用"[ ]"。例如： ${ user. My-Name}应当改为${user["My-Name"]} 2)通过变量动态取值： 如果要动态取值时，就可以用"[ ]"来做，而"."无法做到动态取值。例如： ${sessionScope.user[data]}中data 是一个变量。
