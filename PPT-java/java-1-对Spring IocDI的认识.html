1.IoC/DI是什么

我们经常会听说IoC，也就是Inversion of Controller，控制反转。事实上，IoC并不是一个新鲜的概念，最早在1988年，由Ralph E. Johnson和Brian Foote在论文Designing Reusable Classes中提出。

Dependency Injection，简称DI，即依赖注入。

2.知识剖析

IoC从字面上来说控制反转。

依赖注入(Dependency Injection)是控制反转的一种实现方法。

Spring是一种开源框架，Spring的核心是控制反转(IoC)和面向切面(AOP)。

当前比较知名的IoC容器有：Pico Container、Avalon 、Spring、JBoss、HiveMind、EJB等。

3.常见问题

三连问

能做什么？松散耦合对象

怎么做？使用Spring框架，里边有实现好了的IoC/DI容器

用在什么地方？凡是程序里边需要用到外部资源的情况，都可以考虑使用IoC/DI容器
