1.背景介绍

this是JavaScript语言中定义的众多关键字之一，它的特殊在于它是动态绑定的
this不是定义时确定的,this直接指向它的调用者,
函数每次调用都会产生一个新的上下文环境,this就是这时候确定的,所以说this是动态绑定的.
执行环境(上下文)
函数每次调用都会产生一个新的上下文环境,
函数运行时需要用到的东西，例如var声明、function声明，都在这个上下文环境中寻找。
函数运行完销毁上下文环境。

2.知识剖析

在不同情况下的this...
0.全局作用域
1.函数作为某个对象的成员方法调用
2.函数作为函数直接使用
4.函数作为构造函数调用
5.setTimeout和setInterval以及匿名函数
6.apply()、call()、bind()
demo1
结论:
this总是指向函数的直接调用者（而非间接调用者）；
如果有new关键字，this指向new出来的那个对象；

3.常见问题

//如何正确输出 "Foo function." F = function () { this.myName = "Foo function."; } F.prototype.sayName = function () { var fn = function () { alert(this.myName); } setTimeout(fn, 1000); } var f = new F; f.sayName();

4.解决方案

1、保存this指向
2、使用apply()、call()、bind()处理
3、ES6箭头函数
1.箭头函数作为函数的一种形式,对于this的处理和普通函数有所区别,其没有自己的this上下文,也就是说通过bind/call/apply函数方法设置this值时无效的，会被忽略
2.因为箭头函数没有自己的this上下文，那么当它作为对象的方法函数运行时，this并不指向这个对象
3.箭头函数的的函数体中出现的this在运行时绑定到最近的作用域上下文对象
4.你可以认为箭头函数的this和调用者无关，只和其定义时所在的上下文相关
